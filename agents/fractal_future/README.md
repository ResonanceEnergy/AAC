# Super Agency — FRACTAL FUTURE Swarm Program

## Purpose
Operate the FRACTAL FUTURE research program as an agent swarm under Super Agency governance.
Every cell is fail-local (one cell dying does not cascade). Entropy telemetry flows through all cells.

## Cells
| Cell | Role | Cadence |
|------|------|---------|
| Research | Discover patterns in nature, markets, AI | Daily scan |
| Synthesis | Merge observations into doctrine notes | Weekly |
| Experiment | Design/run/log flagship experiments | On-demand |
| Publishing | Prepare briefs, posts, deck pages | On-demand |
| Governance | Gate reviews, evidence audits, claims checks | Weekly |
| Entropy Sentinel | Monitor entropy across human/system/market | Continuous |

## Playbooks
- `daily_loop.md` — Research cell scan + entropy sentinel check
- `weekly_loop.md` — Synthesis + governance audit + regime map
- `monthly_loop.md` — Doctrine delta + experiment review + publishing queue
- `entropy_controls.md` — Valve activations, drift detection, kill-switch protocol

## Templates
- `ARTIFACT_CONTRACT.md` — Standard contract for any cell output
- `ENTROPY_TELEMETRY.md` — Entropy measurement report format
- `DRIFT_INCIDENT.md` — Incident report when drift/valve trip detected

## Integration
- NCC: Governance cell feeds NCC gate reviews
- AAC: Entropy sentinel feeds AAC regime detection; research cell feeds signal cards
- NCL: Synthesis cell writes to NCL memory layer; governance cell prunes stale notes

## Rules
1. Every cell output MUST carry an evidence label (E0–E4)
2. No cell may override another cell's output — merge conflicts go to governance
3. Entropy sentinel can force-pause any cell if entropy budget exceeded
4. All artifacts versioned and timestamped
