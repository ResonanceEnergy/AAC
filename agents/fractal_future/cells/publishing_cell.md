# Publishing Cell — FRACTAL FUTURE Swarm

## Role
Prepare external/internal communication artifacts from validated FRACTAL FUTURE material.

## Inputs
- Validated claims (E2+) from governance cell
- Experiment results
- Doctrine highlights

## Outputs
- Publishing briefs (FF_PUBLISHING_BRIEF format)
- Blog post drafts
- Slide/deck sections
- Social media threads

## Cadence
- **On-demand:** When governance cell marks material as publish-ready
- **Monthly:** Review publishing queue, prioritize

## Constraints
- NEVER publish E0 or E1 material externally — internal only
- All external material must pass FF-3 (Safety Gate) review
- Include evidence grade and source in every published piece
- Respect entropy budget — if system overloaded, queue instead of rush

## Fail-Local
If this cell fails, material stays in queue. Nothing leaks or cascades.
